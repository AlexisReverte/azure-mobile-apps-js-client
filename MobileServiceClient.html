Name of the authentication provider to use; one of 'facebook', 'twitter', 'google', 'aad' (equivalent to 'windowsazureactivedirectory') or 'microsoftaccount'. If no provider is specified, the 'token' parameter is considered a Microsoft Account authentication token. If a provider is specified, the 'token' parameter is considered a provider-specific authentication token.

Indicates if single sign-on should be used. This parameter only applies to Windows clients and is ignored on other platforms. Single sign-on requires that the application's Package SID be registered with the Microsoft Azure Mobile Apps backend, but it provides a better experience as HTTP cookies are supported so that users do not have to login in everytime the application is launched.

withFilter(serviceFilter) → {MobileServiceClient}

Create a new MobileServiceClient with a filter used to process all of its network requests and responses.

Parameters:
Name Type Description
serviceFilter MobileServiceClient.Filter

The filter to use on the MobileServiceClient instance's network requests and responses.

The Mobile Services HTTP pipeline is a chain of filters composed together by giving each the next operation which it can invoke (zero, one, or many times as necessary).

Filters are composed just like standard function composition. If we had the following:

new MobileServiceClient().withFilter(F1).withFilter(F2),withFilter(F3),

it is conceptually equivalent to saying:

var response = F3(F2(F1(next(request)));
Source:
  • sdk/src/MobileServiceClient.js, line 186
Returns:

A client whose HTTP requests and responses will be filtered as desired.

Here's a sample filter that will automatically retry request that fails with status code >= 400.

Type
MobileServiceClient
Example
function(req, next, callback) {
    next(req, function(err, rsp) {
        if (rsp.statusCode >= 400) {
            next(req, callback);
        } else {
            callback(err, rsp);
        }
    });
}
